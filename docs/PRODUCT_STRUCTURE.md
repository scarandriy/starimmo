# Product Structure

## 1. Public Pages

The initial public site includes:

- Home
- Projects
- Project detail
- Flat detail
- House detail
- About
- Contact

The goal is to establish page hierarchy, reusable sections, and content flow before live data and CRM features arrive.

## 2. Home Page

Purpose:

- Introduce the brand
- Showcase flagship projects and properties
- Build trust quickly
- Drive visitors toward exploration or contact

Suggested sections:

1. Hero with primary value proposition and key CTA
2. Featured projects preview
3. Selected flats and houses preview
4. Trust or service highlights
5. About summary
6. Contact CTA

## 3. Projects Listing Page

Purpose:

- Present active developments in a scannable grid
- Help users compare projects by location and status

Suggested sections:

1. Page intro
2. Optional filter summary area placeholder
3. Project grid
4. Secondary CTA for consultation/contact

## 4. Project Detail Page

Purpose:

- Explain the development
- Show project identity, imagery, and available flats
- Provide confidence through structured information

Suggested sections:

1. Intro block with title, location, construction status, and overview
2. Gallery
3. Project facts and developer details
4. Available flats list
5. Contact CTA

## 5. Flat Detail Page

Purpose:

- Let visitors evaluate a specific apartment inside a project

Suggested sections:

1. Intro block with title, project reference, price, and status
2. Gallery and optional plan image
3. Key facts grid
4. Feature list
5. Project context link
6. Contact CTA

## 6. House Detail Page

Purpose:

- Present standalone homes with stronger focus on lifestyle and land attributes

Suggested sections:

1. Intro block with title, location, price, and status
2. Gallery
3. Key facts grid
4. Outdoor and interior features
5. Contact CTA

## 7. About Page

Purpose:

- Build trust through positioning, market understanding, and service approach

Suggested sections:

1. Intro statement
2. Company approach or values
3. Why clients choose Starimmo
4. Contact prompt

## 8. Contact Page

Purpose:

- Offer a clear way to start a conversation

Suggested sections:

1. Intro statement
2. Contact details and office information
3. Inquiry form placeholder area
4. Response expectations or reassurance copy

## 9. Cross-Page UI Rules

- Every page starts with a consistent introduction pattern.
- Important facts must appear above long descriptive copy.
- CTAs should be present but not repetitive.
- Metadata and URLs must align with the page purpose.
- Placeholder sections must be shaped like future production content, not temporary filler blocks.

## 10. Future Extension Slots

The structure should leave room for:

- property filters
- map-based search
- mortgage tools
- multilingual content
- CRM-powered inquiry capture
- authenticated admin previews
